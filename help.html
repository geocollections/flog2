FLOG2 - Quick start guide

FLOG2 stands for Faunal LOG version 2. It is a small program to save a geologist time and efforts in drawing paleontological range charts and plotting geochemical data from a single section. It allows to set precise scale, plot sample markers and export the drawing to Adobe Illustrator or CorelDraw for final adjustments and combining with other data. FLOG2 is free and open source software, you are free to download, use, modify and distribute the program code, which can be found at github.com/geocollections/flog2.

To start with, use the sample application with no predefined data: geokogud.info/flog2/examples/blank/

Drag the handle in the top center of the window to open the data area where you can copy-paste some text. Once you have meaningful data there then click refresh to draw the graph.

A minimal data matrix that allows producing meaningful chart contains a depth column (with column name "depth") and two rows (= two samples) with numeric sample depth in meters. Program will show the section box with scale and sample markers. For instance:

depth
106.91
110.73

Add one or more parameter columns identified by fossil taxon name or chemical substance to draw charts. For instance:

depthAngochitina longicollisConochitina mamillaCaO [%]
106.91115.6
110.73113.2

Optionally add columns with names "depth_to", "sample_number" and "ID". This allows the program to draw exact vertical extent of samples, plot sample number, and link to database page using sample ID number (only works with samples recorded in Estonian geocollections database). For instance:

depthdepth_tosample_numberIDAngochitina longicollisConochitina mamillaCaO [%]
106.91107.05OM4-32859829115.6
110.73110.84OM4-32559826113.2

Data cells may only contain numbers. Fossil occurrences are commonly coded "1". However, you may use larger numbers to code frequency classes, actual counts of specimens or percentage values. If using negative numbers for fossil occurrences, white markers will be drawn (denoting uncertain identification).

For non-paleontological data most commonly numeric values denoting percentages are used.

Samples may be empty and parameters may also contain no data. For instance:
